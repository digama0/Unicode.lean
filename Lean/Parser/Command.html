def Lean.Parser.Term.quot :
Lean.Parser.Parser

Syntax quotation for terms and (lists of) commands. We prefer terms, so ambiguous quotations like `($x $y) will be parsed as an application, not two commands. Use `($x:command $y:command) instead. Multiple command will be put in a `null node, but a single command will not (so that you can directly match against a quotation in a command kind's elaborator).

def Lean.Parser.Command.terminationHintMany (p : Lean.Parser.Parser) :
Lean.Parser.Parser

A mutual block may be broken in different cliques, we identify them using an ident (an element of the clique) We provide two kinds of hints to the termination checker: 1- A wellfounded relation (p is termParser) 2- A tactic for proving the recursive applications are "decreasing" (p is tacticSeq)
